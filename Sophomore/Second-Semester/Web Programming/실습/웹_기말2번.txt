[문제2] Express , Diango, Rails 프레임워크에 대해서 조사한 후 아래 순서에 따라 각각 정리하세요.
 (5점: ABCD평가)
 1) 정의
 2) 특징

-- Express
- 정의 
Express.js, 또는 간단히 익스프레스(Express) 는 Node.js의 핵심 모듈인 http와 Connect 컴포넌트를 기반으로 Node.js를 빠르고 간편하게 쓸 수 있게 해주는 웹 프레임워크이다.
빠르고 간편하게 쓸 수 있는 이유는 미들웨어(Middleware) 구조 때문입니다. 자바스크립트 코드로 작성된 다양한 기능의 미들웨어는 개발자가 필요한 것만 선택하여 익스프레스와 결합해 사용할 수 있습니다.

- 특징 
Express.js는 HTTP 요청 본문 파싱, 쿠키 파싱, 세션 관리 등을 비롯해 다른 여러 문제를 해결함과 동시에 웹 앱에 MVC 형태의 구조를 제공한다. 
이 같은 앱은 백엔드만 갖춘 REST API에서부터 온갖 기능을 제공하는 고도로 확장 가능한 풀스택(jade-browser와 Socket.IO를 포함하는) 실시간 웹 앱에 이르기까지 다양하다.

-- Django
- 정의
Django는 파이썬으로 만들어진 무료 오픈소스 웹 애플리케이션 프레임워크(web application framework)입니다. 쉽고 빠르게 웹사이트를 개발할 수 있도록 돕는 구성요소로 이루어진 웹 프레임워크랍니다.
다시 발명해야 하는 문제로부터 해방감을 주고요. 새로운 웹사이트를 개발할 때 뒤따르는 간접비용의 부담을 덜어준답니다.

- 특징
Django는 객체 관게 매핑(ORM, Object-Relational Mapping)을 지원하는데 이것은 데이터베이스 시스템과 데이터 모델을 연결시키는 중간 다리 역할인 데이터베이스 연결, 등록, 삭제 등의 쿼리들을 알아서 처리해준다.
Django는 프로젝트를 시작하는 시점에서 기본으로 관리자 화면을 제공해주는데, 이것을 통해 애플리케이션에서 사용하는 데이터들을 쉽게 생성/변경할 수 있어서 별도의 관리 기능을 개발할 필요가 없다.

-- Rails
- 정의
Rails는 프로그래밍 언어인 Ruby로 개발된 오픈소스 웹 프레임워크이며, 완성도 있는 서비스를 빠르게 개발할 수 있는 풀스택 프레임워크이다.
자주 사용하는 개발 방식을 규칙화하고 가치없는 선택지를 제거하는 등 높은 생산성을 가지고 있어서 스타트업 초기 단계에서 프로토타입과 그 이후의 웹 서비스를 만드는 데 주로 사용한다.

- 특징
Rails는 개발자가 설정해야 할 요소를 최대한 줄이고, 이를 규칙으로 미리 제공하여 빠른 개발을 유도한다. 간단히 말하자면 어떤 기능을 개발할 때 한 가지 최선의 모범 답안을 가정하면, Rails는 그 답안을 실행하기 위해 필요한 기본 설정을 제공해준다.
이를 통해 개발자는 정말 필요한 개발 작업에만 집중하여 웹 서비스를 빠르게 구현할 수 있지만 Rails의 규칙에 대해 이해해야 하기 때문에 다른 프레임워크에 비해 초기 진입 장벽이 높다고 느낄 수 있다.
Rails는 사용자가 코드 작성 또는 설정 세팅을 반복하지 않도록 다양한 자동화 옵션을 제공한다. Rails에서 데이터베이스 테이블을 만들면 Rails가 자동으로 스키마 설정을 진행해준다.

-- 공통점
MVC(Model, View, Controller / 하나의 어플리케이션 또는 프로젝트를 구성할 때 그 구성 요소를 3가지 역할로 구분) 패턴, 형태 제공
개발된 언어와 주 목적은 다르지만 사용자의 편리성과 쉬운 개발을 위해 만들어졌다
