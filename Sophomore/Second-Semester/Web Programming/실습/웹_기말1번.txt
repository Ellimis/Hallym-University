 5) 간단한 예를 통한 프로그램 흐름도
 6) 5)번을 수행하기 위해 사용한 소스와 출력결과 화면 

자바스크립트는 웹 브라우저를 제어하는 것이라면 노드 JS 는 자바스크립트를 이용하여 웹 브라우저가 아닌 컴퓨터 자체를 제어한다. 파이썬 자바 php 루비 같은

정의:
	Node.js는 확장성 있는 네트워크 애플리케이션(특히 서버 사이드) 개발에 사용되는 소프트웨어 플랫폼이다.

특징:
	단일쓰레드 이벤트 루프 기반 비동기방식(Non-Blocking I/O)
	- 하나의 쓰레드가 요청을 받으면 바로 다음 처리에 요청을 보내놓고 다른 작업을 처리하다가 먼저 요청한 작업이 끝나면 이벤트를 받아서 응답을 보낸다
	- 동시 요청이 오더라도 처리가 완료될 때까지 기다리지 않아도 되기 때문에 서버 부하가 적다

장점:
	자바스크립트를 동일하게 사용하여 서버단 로직을 처리할 수 있다.
	설치와 개발이 빠르고 쉽다
	Non-blocking I/O와 단일 스레드 이벤트 루프를 통한 높은 처리 성능을 가진다.
	이벤트 기반 비동기방식이라 서버 무리가 적다.

단점:
	이벤트 기반 비동기방식이라 서버단 로직이 복잡한 경우 콜백함수의 늪에 빠질 수 있다.
	단일 쓰레드이기 때문에 하나의 작업 자체가 많이 걸리는 웹 서비스에는 어울리지 않다
	코드가 수행되어야 코드에 에러가 있는지 알 수 있으며, 에러가 날 경우 프로세스가 내려가기 때문에 테스트가 엄청 중요하다.

사용방법:
	https://nodejs.org/en/download/ 에서 각 운영체제에 맞는 installer를 선택하여 node js 를 다운받은 후 터미널이나 cmd에서 node -v 를 쳐서 제대로 설치됬는지 확인을 할 수 있다.
	그래서 터미널이나 cmd에서 node를 치고 실행시켜서 자바스크립트에서 쓰던 문법을 사용할 수도 있고 경로를 js파일이 있는 곳으로 옮겨서 node 파일이름.js를 쳐서 실행시킬 수 있다.
