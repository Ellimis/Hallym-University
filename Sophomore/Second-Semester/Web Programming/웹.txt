웹 프로그래밍 1주차

웹의 구조
 - 인터넷을 활용하여 거미줄처럼 연결된 정보 소통 망, World Wide Web
 - 웹을 사용하기 위해서는 기본적으로 네트워크라는 시스템이 있어야한다.
 - 웹은 종합기술로써 네트워크 시스템을 잘 알아야 한다.

서버는 특정한 분야에서 사용할 수 있는 응용 소프트웨어까지 같이 설치되어 있는 컴퓨터를 웹서버라 한다.

인터넷 서버 위에 웹이라는 새로운 응용 어플리케이션이 작동하는 것
웹은 하나의 응용 어플리케이션, 응용 서비스이다.
인터넷은 네트워크 시스템까지 같이 연결되어 있는 것.
컴퓨터마다 고유한 주소 IP 를 부여하여 컴퓨터 구분, 이를 통해 네트워크에 접속
WWW(World Wide Web) 는 인터넷을  활용하는 응용 서비스 중의 하나이다.

웹 서버 : 클라이언트 쪽이 요청을 하면 그 요청된 자료는 웹 서버가 처리하고 있다
즉 웹 서버 어플리케이션, 프로그램, 소프트웨어가 담당하고 있다

그 외에 필요에 의해 만들어져 있는 새로운 기능들(JSP, PHP 등)로 웹서버용 응용 어플리케이션을 만드는 데
그떄 웹 컨테이너라 한다 즉 동적인 웹 페이즈를 만들고자 하면 웹 컨테이너가 있어야한다

이때 웹 서버와 웹 컨테이너가 같이 포함되어 있는 서버를 웹 어플리케이션 서버라 해서 WAS 라 부른ㄴ다
이때 웹 서버는 단지 클라이언트 쪽에서 요청한 데이터가 있을 때 그 데이터를 임의적으로 뿌려주기만 하는 기법이다
 - 정적인 문서 : 프레젠테이션 로직이라 부른다/

서버 쪽을 통해서 접근해주는 파일들(JSP PHP 등)에 의해서 만들어져 있는 프로그램 로직을 비즈니스 로직이라 부름

이때 프리젠테이션 로직은 웹 서버가 담당
비즈니스 로직은 어플리케이션 서버가 담당 ( 예시 : Tomcat - 아파치 톰캣 서버 )

아파치 톰캣 서버 : 웹을 연동하기 위한 데모닉 읽는 소프트웨어인 아파치 서버를 기점으로 비즈니스 로직인 어플리케이션을 해석할 수 있는 Tomcat 를 같이 쓴다는 얘기

클라이언트를 통해서 서버에 데이터를 요청했을 때 정적인 문서(html 문서, 음악 파일 등)은 프리젠테이션 로직에 의해서 만들어져있기 때문에 웹 서버만 있어도 된다.
그치만 회원가입이나 데이터 베이스에 의해서 원하는 데이터만 추출하고 싶을 땐 js 나 html 로 만들기에 힘드므로 JSP, PHP 등으로 비즈니스 로직을 만들어줘야 사용할 수 있다.
비즈니스 로직이라는 걸 해석하는 소프트웨어가 어플리케이션 서버가 된다

WAS : 웹 어플리케이션 서버. 즉 웹 서버와 웹 어플리케이션 서버를 모두 아우르는 언어다

WAS : Web Server + Web Container
